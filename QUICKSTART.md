# 快速启动指南

## 安装和运行

### 1. 安装依赖

```bash
npm install
```

### 2. 启动开发服务器

```bash
npm run dev
```

### 3. 访问应用

在浏览器中打开：http://localhost:5173

## 功能演示

### 创建任务
1. 点击"新建任务"按钮
2. 填写任务信息：
   - 标题（必填）
   - 描述（可选）
   - 截止时间（可选）
   - 优先级（高/中/低）
   - 重要/紧急标记
   - 标签（用逗号分隔）
3. 点击"保存"

### 查看不同视图

#### 任务列表
- 查看所有任务
- 使用搜索框搜索任务
- 按优先级、标签过滤
- 按创建时间、截止时间、优先级排序

#### 周视图
- 查看一周的任务分布
- 拖拽任务到不同日期修改截止时间
- 使用导航按钮切换周

#### 月视图
- 日历形式查看任务
- 点击日期查看该日所有任务
- 任务数量用彩色点表示（红色=高优先级，黄色=中优先级，灰色=低优先级）

#### 年视图
- 查看12个月的任务统计
- 显示每月任务数量和完成率
- 点击月份进入对应月视图

#### 四象限视图
- 按重要性和紧急性分类任务
- 拖拽任务在象限间移动
- 自动更新重要/紧急标记

### 数据管理

#### 导出数据
1. 点击导航栏的"导出数据"按钮
2. 自动下载 JSON 文件
3. 文件名格式：`todo-backup-YYYY-MM-DD.json`

#### 导入数据
1. 点击导航栏的"导入数据"按钮
2. 选择之前导出的 JSON 文件
3. 确认导入（会覆盖现有数据）

## 开发说明

### 项目结构
- `src/` - 前端源代码
  - `components/` - 可复用组件
  - `views/` - 页面视图
  - `store/` - 状态管理
  - `utils/` - 工具函数
  - `router/` - 路由配置

### 数据存储
- 使用 IndexedDB 存储在浏览器本地
- 数据库名称：`TodoDB`
- 对象存储：`tasks`

### 技术栈
- Vue 3 (Composition API)
- Vue Router
- Pinia
- IndexedDB
- Vite

## 常见问题

### Q: 数据会丢失吗？
A: 数据存储在浏览器本地 IndexedDB 中，除非清除浏览器数据，否则不会丢失。建议定期导出备份。

### Q: 如何清除所有数据？
A: 在浏览器开发者工具中：
1. 打开 Application/应用程序标签
2. 找到 IndexedDB > TodoDB
3. 删除数据库

### Q: 支持多设备同步吗？
A: 当前版本不支持，数据仅存储在本地浏览器。未来可以考虑添加云同步功能。

### Q: 如何自定义样式？
A: 修改 `client/src/style.css` 中的 CSS 变量：
```css
:root {
  --primary-color: #4a90e2;
  --secondary-color: #50c878;
  /* ... */
}
```
